## 🔥 通用踩坑总结

### 坑 1：Excel 大文件性能问题
- **场景**: 10万+ 行数据用 openpyxl 读写
- **解决**: 大文件用 CSV 中间格式，最终报告用 xlsxwriter
- **教训**: 选对工具比优化代码重要

### 坑 2：AI API 调用成本控制
- **场景**: Claude API 批量处理数千条数据
- **解决**: 加本地缓存层（JSON/sqlite），已处理过的跳过
- **教训**: 能用规则解决的不用 AI

### 坑 3：飞书 API 限流
- **场景**: 批量同步到飞书多维表格
- **解决**: 延时 + 分批 + 失败重试
- **教训**: 第三方 API 必须考虑限流策略

## 🧠 我的设计原则

1. **最简可用先行**：不为了"以后可能"的需求加复杂度
2. **AI 是工具不是银弹**：把 AI 预算花在刀刃上
3. **数据管道要幂等**：脚本必须能重复运行不产生副作用
4. **内部工具不追求完美 UI**：功能可用 > 界面精美
