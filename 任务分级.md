# 🔀 任务分级细则

> AI 收到任务后，自动按此表分级并采取对应行动。

---

## 🟢 A 级 — 简单任务（直接干）

### 典型场景
- 修改单文件中的几行代码（修 bug、改文案、加注释）
- 添加一个已有模式的组件/函数（如照着已有的再写一个）
- 执行已知的 shell 命令（npm install、git status）
- 回答一个事实性问题（这个函数干什么的？）
- 格式化、lint 修复
- 追加/修改一句对话日志

### 执行方式
- 不读多余的 md（`工作规范.md` 除外）
- 不等确认，直接改
- 事后一句话汇报结果
- 验收可省略（抽查）

---

## 🟡 B 级 — 中等任务（读 md → 规划 → 干）

### 典型场景
- 新增一个完整功能（前端页面 + API）
- 修改涉及 3 个以上文件
- 数据库表结构变更
- 技术选型（用 A 库还是 B 库？）
- 代码重构（较大范围）
- 数据分析脚本编写

### 执行方式
1. 读取相关 md（`我的项目.md`、`最佳案例.md`、`项目经验.md`）
2. 执行 CEO 拆解：3-5 行规划
3. 驾驶舱对齐标准
4. 执行
5. 验收回写

---

## 🔴 C 级 — 复杂/全新任务（确认 → 方案 → 干）

### 典型场景
- 引入新技术栈
- 跨模块重构（影响 >10 个文件）
- 架构级变更
- 从未做过的新领域（不在我的工作领域列表中）
- 涉及安全、权限、支付等敏感操作
- 部署/CI/CD 变更
- 删库、迁移等不可逆操作
- 我自己摸不准需要你帮判断的

### 执行方式
1. **停！先跟我确认**
2. 执行 CEO 完整拆解：问题域、边界、方案选项
3. 展示方案（含备选、trade-off），等我点头
4. 同意后执行，每完成一个里程碑汇报
5. 完整验收 + 经验回写 + 品味沉淀

---

## 🎯 快速自查（AI 自问）

收到任务后，AI 问自己三个问题：

1. **我做过类似的事吗？** → 否 → C 级
2. **影响范围 ≤ 3 个文件？** → 否 → B 级起
3. **有现成模板/模式可抄？** → 否 → B 级起

三个全 Yes → A 级，直接干。
