# 达人库分析系统

## 项目概览

- **仓库**: `~/Documents/Claude/Projects/达人库/`
- **时间**: 2025 ~ 至今
- **类型**: 数据处理脚本集
- **状态**: 维护中
- **我的角色**: 数据分析 + 脚本开发
- **团队规模**: 1 人

## 技术栈

Python/pandas + openpyxl + Instagram API + TikTok API + Modash 数据

## 项目简介

多数据源达人聚合分析系统。从 Modash、Instagram、TikTok 采集达人数据，进行 Bio 分类、邮箱补全、聚类分析、背书策略分析，输出营销报告和建联分层表。服务 IM8 / WonderBiotics 品牌。

## 我负责的部分

- 20+ 数据处理脚本编写
- 达人 Bio 分类算法（AI 驱动）
- 缺失邮箱补全（API + 规则匹配）
- 聚类分析和分层策略
- Excel 报告自动生成

## 技术难点 & 解决方案

- **多源数据去重**：Instagram、TikTok、Modash 数据源的达人 ID 体系不同，通过用户名+Bio 相似度做模糊匹配去重
- **Bio 分类**：用 Claude API 对达人 Bio 进行自动分类（Fitness / Lifestyle / Beauty 等），处理多语言 Bio
- **邮箱补全**：先尝试 API 查询，失败后用规则推断（名+姓+域名组合）

## 踩坑记录

- Excel 大文件（10万+行）用 openpyxl 写入极慢，改用 xlsxwriter 或用 CSV 中间格式
- API 调用费用控制：Claude API 批量处理 Bio 分类时容易超预算，需要做缓存（已处理过的达人跳过）

## 收获 & 思考

- pandas 的 `apply()` 配合 AI API 调用可以做很强大的数据增强
- 数据清洗脚本要考虑幂等性（重复运行不出错）
- 分层报告比原始数据表格对业务更有价值
