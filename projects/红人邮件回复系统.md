# 红人邮件回复系统（WonderBiotics KOL CRM）

## 项目概览

- **仓库**: `~/Documents/Claude/Projects/红人邮件回复系统/`
- **时间**: 2025 ~ 至今
- **类型**: Web 应用（本地 CRM）
- **状态**: 维护中
- **我的角色**: 全栈开发
- **团队规模**: 1 人

## 技术栈

Python FastAPI + SQLAlchemy + aiosqlite + Jinja2 模板 + Claude API + 飞书 SDK

## 项目简介

为 WonderBiotics 品牌构建的 KOL 邮件建联自动化 CRM。三栏布局工作台（左：收件箱 / 中：邮件正文+AI 建议 / 右：达人信息面板），支持多邮箱账号、批量发送（带像素追踪）、里程碑自动推进（M01→M23）。

## 我负责的部分

- 全部后端 API（96KB 的 `routes/api.py`）
- 前端 Jinja2 模板渲染（SSR 风格）
- AI 辅助回信功能（Claude API 根据达人信息生成个性化回复建议）
- 邮件追踪像素实现（1x1 透明图片 + 唯一 ID）
- 飞书多维表格同步（达人状态回写）
- 多邮箱 IMAP 收件和 SMTP 发件管理

## 技术难点 & 解决方案

- **多邮箱并发管理**：用 aiosqlite 存储多账号凭证，asyncio 并发收取多邮箱，避免阻塞
- **邮件追踪**：嵌入 1x1 透明像素图片，URL 带唯一 tracking ID，后端记录打开时间
- **AI 回信上下文**：将达人 Bio、历史邮件、品牌信息拼成 prompt，让 Claude 生成个性化回复
- **里程碑自动推进**：解析邮件内容关键词和用户手动标记，自动推进 M01→M02→...→M23

## 踩坑记录

- aiosqlite 并发写入冲突：高频邮件收件时偶尔出现 database locked，加了重试机制和写入队列解决
- 飞书多维表格 API 限流：批量同步时触发限流，需要加延时和分批处理

## 收获 & 思考

- aiosqlite 适合本地单机场景，但并发高时还是 PostgreSQL 更稳
- Jinja2 做 SSR 虽然传统，但对于这种内部工具来说开发效率高，不需要前后端分离
- AI 辅助回信极大提升了建联效率，但需要人工最终确认（Human-in-the-loop）
