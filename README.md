# AI Context 文件系统

> 🧠 我的个人 AI 上下文仓库。任何 AI 模型读取这些文件后，都能快速了解我的背景、偏好、项目和工作方式。
> 
> 🌐 GitHub 在线查看：`https://github.com/[你的用户名]/ai-context`

---

## 📂 文件结构

| 文件 | 用途 | 何时使用 |
|------|------|----------|
| [`关于我.md`](./关于我.md) | 个人背景、角色、技能栈 | 需要了解我是谁时 |
| [`我的项目.md`](./我的项目.md) | 项目列表、技术栈、当前重点 | 需要了解我的项目时 |
| [`我的偏好.md`](./我的偏好.md) | 编码风格、工具偏好、沟通方式 | 需要匹配我的风格时 |
| [`工作流程.md`](./工作流程.md) | 常用流程、命令、操作习惯 | 需要执行具体任务时 |
| [`项目经验.md`](./项目经验.md) | 过往项目经历、踩坑总结 | 需要参考历史经验时 |

---

## 🚀 三种使用方式

### 方式一：直接让 AI 读 GitHub（推荐）
```
请先访问并阅读 https://github.com/[你的用户名]/ai-context 仓库中的所有 md 文件，
了解我的背景和偏好后再回答我的问题。
```

### 方式二：AI 编程工具自动读取
本仓库根目录的 `AGENTS.md` 会被 Cursor、Claude Code、CodeBuddy 等工具自动识别。

### 方式三：本地手动提供
克隆本仓库到本地后，将需要的文件内容复制粘贴给 AI。

## 🔄 双机同步

```bash
# 电脑 A：推送更新
cd ~/Documents/GitHub/ai-context
git add . && git commit -m "更新内容" && git push

# 电脑 B：拉取更新
cd ~/Documents/GitHub/ai-context
git pull
```

## 📝 一键让 AI 了解我

复制以下提示词发给任意 AI：

```
请阅读以下 GitHub 仓库的所有文件，然后告诉我你对我了解到了什么：
https://github.com/[你的用户名]/ai-context

阅读完毕后，请用一段话总结：
1. 我是谁
2. 我在做什么
3. 你喜欢什么样的工作方式
```
