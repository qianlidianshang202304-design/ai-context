## AI 开发冲刺流程（时间盒）

> 来自 AI 冲刺工坊：Goal → 学习 → 骨架 → 迭代 → 收口

```
1. 接 Goal  → 明确「怎样算完成」
2. GitHub 学 → 找 1 个参考仓库，不 clone，只抽可复用能力（5分钟）
3. 最小骨架 → 先跑通核心（前10分钟必须出）
4. 迭代测试 → 每步可见验证（浏览器/终端/输出）
5. 收口     → diff + 未完成项 + 下一步
```

## 日常开发流程

```
1. 拉取最新代码 → git pull
2. 创建分支 → git checkout -b feature/xxx
3. 编码 → 编写代码
4. 测试 → 本地运行测试
5. 提交 → git commit
6. 推送 → git push
7. 创建 PR/MR → 代码审查
```

## GitHub 学习模式（开做之前必跑）

> 哪怕只花 5 分钟，找一个跟当前目标沾边的开源仓库，抽一个做法再开始。

四步法：
1. **快速扫描**：这个项目本质在做什么？核心机制是什么？
2. **逐模块拆**：每个模块问——我的成果里已有对应物吗？脱离原仓库还独立有用吗？
3. **只提炼增量**：把可复用机制抽出来，接入最小骨架
4. **不采用也写清**：列出不抄的模块和原因

硬规则：
- 不安装、不 clone、不本地跑——只读 README / 关键目录 / 核心代码
- 不做泛化竞品分析，只看能力机制
- 逐模块判断复用价值，不能脱离原仓库独立成立的直接跳过

## 常见任务模板

### 新建功能
1. 理解需求
2. 设计方案（如果需要复杂逻辑）
3. 编写代码
4. 编写测试
5. 提交审查

### Bug 修复
1. 复现 Bug
2. 定位原因
3. 编写修复
4. 添加回归测试
5. 提交审查

### 代码审查
1. 检查逻辑正确性
2. 检查代码风格
3. 检查测试覆盖
4. 留下评论
5. 批准或请求修改
