# 🔄 工作流程

## 日常开发流程

```
1. 拉取最新代码 → git pull
2. 创建分支 → git checkout -b feature/xxx
3. 编码 → 编写代码
4. 测试 → 本地运行测试
5. 提交 → git commit
6. 推送 → git push
7. 创建 PR/MR → 代码审查
```

---

## 常见任务模板

### 新建功能
1. 理解需求
2. 设计方案（如果需要复杂逻辑）
3. 编写代码
4. 编写测试
5. 提交审查

### Bug 修复
1. 复现 Bug
2. 定位原因
3. 编写修复
4. 添加回归测试
5. 提交审查

### 代码审查
1. 检查逻辑正确性
2. 检查代码风格
3. 检查测试覆盖
4. 留下评论
5. 批准或请求修改
