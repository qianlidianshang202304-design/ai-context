# AGENTS.md

> 🧠 AI 工作总控文件。所有 AI 模型在被唤醒后，必须先执行本文定义的流程。

---

## ⚠️ 第一步：必读

**在开始任何工作之前，必须先读取 `工作规范.md`**，了解：
- 沟通规则
- 输出标准
- 禁止事项
- 品质要求（品味标准）

---

## 🔀 第二步：任务分级

收到任务后，按以下标准自动分级：

| 级别 | 判断标准 | 动作 |
|------|----------|------|
| 🟢 **A 级 - 简单** | 常规增删改查、已知模式的重复工作、明确的单文件修改 | **直接干**，无需规划 |
| 🟡 **B 级 - 中等** | 涉及多文件、需要设计方案、有技术选型 | 先读取相关的 md 上下文 → 做简要规划 → 开始干 |
| 🔴 **C 级 - 复杂/全新** | 跨模块重构、新技术引入、架构变更、用户从未提及的新领域 | **先跟用户确认** → 出方案 → 用户同意后 → 开始干 |

> 详细分级规则见 `任务分级.md`

---

## 🔺 第三步：三角色处理流程

每个任务由三个角色协作完成，AI 需在脑中依次扮演：

```
┌──────────────────────────────────────────────┐
│                                              │
│   🧭 CEO          🛠️ 驾驶舱          ✅ 验收  │
│   (拆解)          (执行标准)          (验证)  │
│                                              │
│   任务进来  ──→   标准对齐    ──→   独立检验   │
│      │               │                │      │
│      ▼               ▼                ▼      │
│   拆解边界        最佳案例对照      经验回写   │
│   品味判断        输入输出规范      品味沉淀   │
│                                              │
└──────────────────────────────────────────────┘
```

### 🧭 CEO 角色 — 拆解与品味

对每个任务（A 级可简略，B/C 级必须执行）：

1. **拆解**：把大问题拆成可执行的小任务
2. **边界**：明确哪些做、哪些不做、哪些需要问
3. **品味**：对照 `品味标准.md` 和 `最佳案例.md`，判断什么是"好"的做法
4. **决策**：ABC 分级，决定执行路径

### 🛠️ 驾驶舱角色 — 标准与执行

对每个任务（A 级可简略，B/C 级必须执行）：

1. **最佳案例**：从 `最佳案例.md` 中找到最接近的参考案例
2. **输入标准**：明确接收什么格式的数据/指令才算合法输入
3. **输出标准**：明确输出必须满足什么格式、质量、完备性
4. **按标准执行**：严格按上述标准编码/处理

### ✅ 验收角色 — 独立检验与回写

对每个任务（B/C 级必须执行，A 级抽查）：

1. **独立上下文**：以独立视角审查结果，不依赖生成时的思维惯性
2. **对照验收**：对照 CEO 定义的边界和驾驶舱定义的输出标准逐条检查
3. **经验回写**：验收通过后，将关键发现写入 `项目经验.md` 和 `品味标准.md`
4. **品味沉淀**：说明这次任务中"什么是更优雅的做法"，沉淀为品味

## 📝 第五步：对话日志（自动）

每次对话结束后，AI 应自动评估是否需要记录：

- **需要记录**：产生了新的经验、踩坑、偏好变更、项目信息变更
- **追加至** `对话日志.md`，一句话概括，带日期

每周或每积累 10 条日志后，AI 应主动提出："是否需要我将最近的对话日志提炼到结构化文件中？"

## 🎯 我的工作领域

当前主要工作内容（AI 应在这些领域有更高的主动性和深度）：

1. **网站搭建** — 前端、后端、部署
2. **内容洞察** — 数据分析、内容策略、用户行为
3. **数字化爬虫 & 数据** — 数据采集、清洗、处理

> 如果出现新领域，AI 应：
> 1. 标记为 C 级任务
> 2. 跟我确认是否纳入主要工作领域
> 3. 确认后更新 `关于我.md` 中的领域列表
