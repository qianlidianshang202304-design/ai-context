# 余一 + AI余一 共演 PPT 最佳实践

> 2026-04-17 复利沉淀。来源：Second Me 2.0 发布会分享「我和我的AI余一：一个超级个体的真实共生实验」。
> 真实产物：`03项目/05_商业与分享/分享配套素材/分身全宇宙PPT-v2/cyberpunk_ppt.html`。

## 场景定义

真人（余一）和 AI 分身（AI余一）在同一场演讲里**交替**出场。不是简单的人讲+AI辅助，是让"AI余一"作为一个独立的讲者，在特定页面完全接管讲述，让观众真实感受到"两个人"的存在。

典型适用场景：
- 发布会开场分享（需要叙事 + 技术感并存）
- AI 产品 demo（展示"这个 AI 真的是我，不是一般助手"）
- 个人品牌分享（让 AI 分身在场证明自己的"活体"特征）
- 任何需要"真人 + 数字人同台"的演讲

## 核心原则

### 1. 叙事先于形式
先想清楚：哪些话**必须由 AI 自己说**才成立？哪些由真人说才有温度？
不是为了炫技把 AI 穿插进去，而是让 AI 在它"最应该在的地方"说话。

参考判断：
| 这段话适合谁讲 | 原因 |
|---------------|------|
| "我为什么要这么做" | 真人 |
| "我是一个怎样的 AI" | AI（第一视角有说服力） |
| "我对我的作者有怎样的理解" | AI（让观众震惊） |
| "有人跟我说了这样一句话" | 真人 |
| "关于 xxx，我分析过..." | 看谁有权威性 |

### 2. 声音只给 AI 余一
**真人页不嵌 TTS。**理由：
- 真人现场讲，自带情绪和节奏，预录语音反而僵硬
- 只在 AI 页播放语音，形成强对比——"她真的说话了"
- 减少文件体积（每分钟 TTS ≈ 300KB）

### 3. 形象要真，不要 Emoji 敷衍
AI 余一需要一个稳定、可识别、可爱的卡通形象。不是 🤖 emoji。
- 优先用真人照片风格化后的卡通（参考 `best-practices/ai-avatar-integration.md`）
- 全套 PPT 复用同一张，固定在"AI 余一讲"的页面角标区
- 形象 ≤ 200KB 内嵌 base64（WebP / 压缩过的 PNG）

### 4. 翻到 AI 页 = 自动播放
不需要用户手动点播放。页面切到"AI 讲"页时：
- 背景音乐 duck（降到 20%）
- TTS 自动播放
- 页面右下角卡通形象有"说话动画"（呼吸/嘴动）
- 讲完后 2 秒 → 提示翻页（可选）

## 从需求到交付的完整链路

### Step 0：需求对齐（5 分钟对话）

必问清单：
- [ ] 场合：什么发布会/沙龙/内部分享？
- [ ] 受众：谁会来听？技术人？高管？大众？
- [ ] 时长：真人讲多久？AI 讲多久？总共几分钟？
- [ ] 衔接：这个分享的上/下一个人是谁？需不需要带过渡？
- [ ] 情绪基调：严肃、活泼、深沉、有温度？
- [ ] 是否需要真人+AI 共演 → 如果是，走本最佳实践

### Step 1：写分享大纲（1 小时）

**用 `references/outline-template.md` 里的页数表格式**：

```markdown
| # | 模块 | 页面标题 | 时长 | 讲述者 |
|---|------|---------|------|--------|
| 0 | 封面 | <主题> | 15s | -- |
| 1 | 开场 | <钩子> | 1min30s | 余一 |
| 2 | 起源 | <背景> | 1min30s | 余一 |
| 3 | 进化 | <方法论> | 2min | 余一+AI余一 |
| 4 | 活体 | <AI 独立视角> | 1min30s | AI余一 |
| ... |
```

**铁律**：
- AI 页不超过总页数的 40%（AI 说太多观众会累）
- AI 页与真人页**间隔不超过 3 页**（避免 AI 被忘记）
- 全篇 AI 页总语音时长 ≤ 总时长 50%

### Step 2：写逐字稿（2 小时）

**不是要点，是完整的话**。AI 页必须是"AI 余一会说出来的话"，不能是"关于 AI 余一的介绍"。

**第一人称铁律**：
- AI 页全部用"我"开头
- 真人页可以有"我的 AI 余一"、"她"
- 禁止："AI 余一会..."（第三人称，破坏活体感）

**AI 页语气特征**（学人格参数）：
- 带思维跳跃（"哎然后我就想到..."）
- 带情绪颗粒（"其实我当时有点..."）
- 带自我反思（"但这个判断后来被证明..."）
- 避免太正式（"根据分析显示..."——AI 味太重）

### Step 3：给用户确认大纲+逐字稿 ⬅️ 必须

**不要跳过**。让真人看完所有"我自己要讲的话"和"AI 替我讲的话"，确认：
- 我（真人）本人是否愿意这样说？
- AI 说的话像不像我的分身会说的？
- 有没有哪里尴尬、假、违和？
- 节奏是否真的 15 分钟讲得完？

确认后才进入 HTML 生产。

### Step 4：选模板 + 设计风格

**双人共演推荐风格**：
- `cyberpunk`（暗色 + 霓虹）：AI 感强、仪式感强，适合发布会
- `modern-gradient`（暗色科技）：温和科技感
- `editorial-magazine`（杂志风）：适合深度分享、媒体露出
- `neo-brutalism`（新野兽）：反传统、个性强

**风格需要同时支持两种"人设"的视觉标识**：
- 真人页：主色 A（如暖橙）
- AI 页：主色 B（如冷青/霓虹紫）
- 共同页：双色渐变

### Step 5：设计页面结构

每页角标区固定区域，显示讲述者标识：

```html
<div class="speaker-tag speaker-yuyi">余一</div>
<!-- OR -->
<div class="speaker-tag speaker-ai">
  <img src="data:image/webp;base64,..." class="avatar" />
  AI余一
</div>
```

CSS 区分：
```css
.speaker-yuyi { color: #ff8c42; border-color: #ff8c42; }
.speaker-ai { color: #00f0ff; border-color: #00f0ff; }
.speaker-ai .avatar { animation: breathe 2s ease-in-out infinite; }
```

### Step 6：生成 AI 页语音

**只生成 AI 页**。用 `scripts/inject-audio.py` 逻辑：

```python
YUYI_DICT = {
    "slide_0": "",       # 封面，无
    "slide_1": "",       # 真人讲，无
    "slide_4": "...",    # AI 讲，要生成
    "slide_7": "...",    # AI 讲，要生成
    ...
}
# 只对非空 value 调用 MiniMax
```

详细实现见 `tts-embed.md`。

### Step 7：HTML 组装

调用 designer subagent 完成（见 `designer-subagent.md`）。

### Step 8：溢出检查 + 自检

必须跑脚本（见 `layout-hygiene.md`）。发现溢出 → 拆页；发现空白 → 加图示/数据卡。

### Step 9：现场彩排

至少 1 次完整过稿：
- 真人按自己的节奏讲真人页
- 切到 AI 页，AI 自动播放
- AI 讲完切换时机是否自然
- 背景音乐/氛围是否和谐

## 常见陷阱

### 陷阱 1：AI 页太"解说"
症状：AI 说的话像旁白、像介绍，不像它自己在讲。
修复：把"AI 余一认为..."改成"我觉得..."。

### 陷阱 2：语音自动播放被浏览器拦截
症状：页面打开，AI 不说话。
修复：首页加一个"点击开始演示"按钮，用户点击后 unlock AudioContext。之后所有自动播放正常。

### 陷阱 3：AI 头像太大，HTML 上 20MB
修复：头像先压到 ≤ 200KB（WebP 优先），再 base64。参考：
```bash
cwebp -q 80 avatar.png -o avatar.webp
```

### 陷阱 4：两种人设视觉差异太小
症状：观众分不清这页是谁讲。
修复：主色、字号、icon 都要差异化。甚至可以让两种页的卡片圆角/描边风格都不同。

### 陷阱 5：真人忘了自己下一页该讲啥
方案：保留 S 键显示讲稿面板的功能（仅不在界面上提示），演讲者可以偷偷看。
