# Demo Deck · 三页展示生成

## 产出物

读取参与者的成果，产出三样东西：

1. **3 页 HTML PPT**（单文件，可断网打开，可翻页）
2. **分享策略**（60 秒怎么讲、讲给谁、想要什么反馈）
3. **口播提示卡**（每页一句要说的话 + 一个停顿点）

## 不负责

- 不做开发（→ `ai-dev-sprint`）
- 不写完整长 deck（这是三页展示版，不是报告）
- 不替用户判断项目好坏（→ 段 6 `pick-top3`）
- 不知道现在在工坊哪一段（→ `sprint-guide`）

## 输入契约

必需其一：成果目录路径 / 仓库 / 链接 / 文档 / 截图。
可选：受众、想要的反馈类型、想组队的方向。

缺输入时只问改路由的：「成果在哪、给谁看」。

## 执行链

| 步  | 动作       | 产出                                           |
| --- | ---------- | ---------------------------------------------- |
| 1   | 读成果     | 抓「做了什么 / 核心亮点 / AI 在哪发力 / 卡点」 |
| 2   | 定三页骨架 | 见下方三页结构                                 |
| 3   | 写分享策略 | 60 秒脚本 + 受众 + 求助点                      |
| 4   | 生成 HTML  | 单文件，继承视觉系统                           |
| 5   | 自检       | 契约 + 句式 + 一页一判断                       |

## 三页结构（固定）

| 页  | 核心判断             | 视觉承载                                            |
| --- | -------------------- | --------------------------------------------------- |
| P1  | 我做了什么           | 大标题 + 成果截图 / 链接二维码 / 关键数字           |
| P2  | 我怎么做的 + AI 杠杆 | 路径流 or 前后对比（AI 加速了哪一步）               |
| P3  | 感受 + 想和谁组队    | 一句感受 + 想要的互补能力（编码关系，不是 card 墙） |

## 视觉硬门（继承工作区可视化标准）

- 一页一判断 + 一个视觉承载，禁止纯文字卡片墙
- 中文正文/标签用 sans（PingFang SC / Noto Sans SC）；serif 只用于大引语
- 0 inline font-size；六色锁内单色相锚点；无装饰色条
- 禁用句式：不是…而是… / 破折号装饰 / emoji
- 单文件，键盘翻页 + 进度条 + 页码
- 参考（包内本地副本，别人电脑上也能用）：`references/BP031-html-ppt-presentation.md`
- 资源内嵌（断网可开）：`references/embed-everything.md`

## TTS（可选，仅 AI 自述页）

需要语音解说时：
- 只给「AI 视角自述」页配 TTS，真人讲的页不配
- MiniMax 返回 hex，需 `bytes.fromhex` 再 base64 内嵌
- 1 分钟 mp3 ≈ 300-500KB，三页可接受
- 翻到 AI 页自动播放
- 完整机制：`references/tts-copresent.md`

## 分享策略模板

```text
受众：__（技术 / 业务 / 评委）
60 秒脚本：
  0-15s 这是什么 + 给谁用
  15-40s 演示核心（打开链接 / 跑一下）
  40-55s AI 帮我做到了 __
  55-60s 我想找 __ 一起把它做成 __
求助点：我卡在 __，希望队友能 __
```

## 验收

- [ ] 三页齐全，每页一个判断 + 一个视觉承载
- [ ] HTML 单文件可断网打开、可翻页
- [ ] 分享策略包含受众 + 60 秒脚本 + 求助点
- [ ] 口播提示卡每页一句话
- [ ] 句式 / emoji / serif 正文门通过
