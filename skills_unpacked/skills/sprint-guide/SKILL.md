# sprint-guide · AI 冲刺工坊向导

> 这是参与者进工坊后**第一个读的 skill**。它不干活，只做一件事：告诉你现在在哪一段、
> 该用哪个 skill、这一段要交出什么。每段都按同一个节奏推进——先做出第一版，拿它换反馈，再优化。

## 贯穿全程的一条铁律：干中学 · 第一版优先

工坊不是听课，是动手。每一段都走同一个微循环，不要在任何一段停在「想清楚再做」：

```
出第一版（哪怕粗糙、不完整）
   -> 拿第一版换反馈（AI 自检 / 现场 Demo / 队友意见）
   -> 基于反馈优化下一版
```

- 卡住超过几分钟 = 你想得太多。先让 AI 出一个能看的东西，再在它上面改。
- 「不完美但能展示」永远优于「完美但还没出来」。
- 每段到点就切，带着半成品进下一段，下一段会让它变好。

## 七段流程地图

| 段  | 名称            | 时长 | 你要交出什么                       | 用哪个 skill                                          |
| --- | --------------- | ---- | ---------------------------------- | ----------------------------------------------------- |
| 1   | 学习和明确      | 30m  | 一个可验证的 Goal                  | `ai-first` -> `codex-goal`（`skill-onboarding` 选读） |
| 2   | 个人 Build      | 30m  | 一个最小可展示成果                 | `ai-dev-sprint`                                       |
| 3   | 第一次 Demo     | 60m  | 三页 demo + 想组队的方向           | `demo-deck`                                           |
| -   | 茶歇 & 组队     | 15m  | 组队意向 + Team Brief              | `team-match`                                          |
| 4   | 组队 Build      | 60m  | 团队版成果（在 Demo 1 基础上优化） | `ai-dev-sprint` + `team-match`                        |
| 5   | 第二次 Demo     | 30m  | 团队 demo + 24 小时动作            | `demo-deck`                                           |
| 6   | 点评 & 挑 Top 3 | -    | 选出值得继续的项目                 | `pick-top3`（主持人侧）                               |

## 按段路由 · 每段怎么走

### 段 1 · 学习和明确（30 分钟）

目标：从「我不知道做什么」到「我有一个能验证的 Goal」。

1. （选读，约 5 分钟）想理解 skill 是什么、AI 怎么用得好 -> `skill-onboarding`。已经懂的跳过。
2. 挑活儿（约 10 分钟）-> `ai-first`：从你反复做的事里，圈一个今天能做出 demo 的最小切片。
3. 写 Goal（约 15 分钟）-> `codex-goal`：把切片写成陈述句终态 + 验收 + 边界的可粘贴 Goal。

第一版优先：Goal 先写出来再打磨，别为措辞纠结。能让 AI 看懂「做什么、怎样算完成」就够开工。

### 段 2 · 个人 Build（30 分钟）

目标：把 Goal 变成一个能打开 / 能跑的东西。-> `ai-dev-sprint`

- 前 10 分钟必须让 AI 出第一版可展示骨架，不要先查全资料。
- 同一个错误重试不超过 3 次，换问法或缩小 Goal。
- 每个能跑的版本立刻保存，防崩溃归零。
- 到点停。带着半成品进 Demo，下一段会让它变好。

### 段 3 · 第一次 Demo（60 分钟）

目标：60 秒讲清楚，并找到想组队的方向。-> `demo-deck`

讲三件事：做了什么 / 有什么感受（AI 帮了你什么、卡在哪）/ 想和什么样的人组队。
这一段的反馈（别人的提问、想合作的信号）就是段 4 优化的输入，记下来。

### 茶歇 & 组队（15 分钟）-> `team-match`

把 Demo 里听到的互补点，变成具体邀请。每个团队产出一张 Team Brief：团队名 / 做什么 / 谁负责什么 / 60 分钟后的 Demo 长什么样。

### 段 4 · 组队 Build（60 分钟）

目标：在 Demo 1 的成果和反馈基础上，做出更完整的团队版。-> `ai-dev-sprint` + `team-match`

这一段是「再优化」的主战场，三条路任选：
- 深化：Demo 1 的方向有人接力，补设计 / 数据 / 部署 / 文档。
- 合并：两个互补成果拼成一个产品雏形。
- 开新题：Demo 1 激发的新想法，快速验证。

一人牵头 Demo，其他人并行补位，AI 是团队的第 N+1 个成员。

### 段 5 · 第二次 Demo（30 分钟）-> `demo-deck`

讲三件事：团队版比 Demo 1 多了什么 / AI 在哪一步产生了真实杠杆 / 如果继续，24 小时内第一件事做什么。

### 段 6 · 点评 & 挑 Top 3（主持人侧）-> `pick-top3`

按成果完整度 / AI 杠杆可见 / 24h 动作可信三维，选出值得继续推进的项目。

## 我现在该用哪个 skill（快速自查）

- 不知道做什么 -> `ai-first`
- 知道做什么但没写 Goal -> `codex-goal`
- 有 Goal 要开始做 -> `ai-dev-sprint`
- 做出东西了要展示 -> `demo-deck`
- 要组队 / 找队友 -> `team-match`
- 想评一个 skill 好不好 -> `skill-eval`（工具，按需）
- 主持人要挑 Top 3 -> `pick-top3`

## 验收（这个向导用对了吗）

- [ ] 参与者任何时候都能说出自己在第几段、下一步用哪个 skill
- [ ] 每段都遵守「先出第一版，再换反馈优化」，没有人卡在空想
- [ ] 段与段之间无缝交接（上一段的产出是下一段的输入）
