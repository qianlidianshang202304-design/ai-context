# 周更 Hook：AI 能力快照自动刷新（C 混合模式）

> 模式：C 混合 = 自动跑搜索 + 待确认才合并

---

## 触发时机

- **自动触发**：每周日凌晨（夜间守护模式之外的时段）
- **手动触发**：用户说"刷新 AI 能力快照" / "更新快照"

---

## 流程

### Step 1：检查是否需要刷新

读 `snapshots/ai-capability-snapshot.md` 的"上次更新"日期：
- ≤ 7 天 → 跳过本次
- > 7 天 → 进入 Step 2

### Step 2：跑搜索

按快照表的能力维度，每个维度调用 `.yuyi/skills/web-access` 跑：

```
搜索清单（每个维度）：
1. "[维度英文] best model 2026 [当前月份]"
2. "[维度中文] 最强 2026 [当前月份]"
3. 抓 Artificial Analysis / LMArena / 各厂商 changelog 主页
```

时间窗：3 个月内。

### Step 3：写到 pending

把搜索结果写到 `snapshots/pending/[YYYY-MM-DD]-update.md`，格式：

```markdown
# AI 能力快照更新建议 - YYYY-MM-DD

## 文本生成
- 旧值：[当前快照]
- 新值（建议）：[搜索结果]
- 来源：[URL @ 日期]
- 变化原因：[新模型发布 / 排名变化 / 价格变化]

## 代码生成
...
```

### Step 4：写异步沟通区

在 `meta/异步沟通.md` 加一条：

```
## AI 能力快照更新待确认（YYYY-MM-DD）

本周自动搜索发现 N 处可能的能力变化，已写到：
`.yuyi/skills/ai-first-redesign/snapshots/pending/[YYYY-MM-DD]-update.md`

主要变化（前 3 条）：
1. ...
2. ...
3. ...

请审核 → 确认后我合并到主快照。
```

### Step 5：等用户确认

用户审核后回 "确认" / "改 X 项后确认"：
- 全确认 → AI 合并 pending 到主快照，更新"上次更新"日期，归档 pending 文件到 `pending/archived/`
- 部分确认 → 按用户指示修改后再合并
- 全否决 → 标注"本周更新被否决"，pending 文件归档但不合并

---

## 失败兜底

如果某个维度 web-access 搜不到合理结果：
- 不要乱填
- 在 pending 里标注"[本周搜索未发现明确变化]"
- 主快照保持上一版

---

## 反作弊

- ❌ 没真的调 web-access，凭印象更新
- ❌ 直接合并到主快照，跳过用户确认
- ❌ pending 里没附 URL

---

## 安装说明

第一次启动这个 skill 时，**必须先触发一次手动刷新**，因为初始版快照是草拟的未经验证的占位符。

启动指令：
> 用户：开始拆我的 X
> AI：检测到 AI 能力快照为初始草拟版，先跑一次完整刷新，约 X 分钟。开始 / 跳过？
