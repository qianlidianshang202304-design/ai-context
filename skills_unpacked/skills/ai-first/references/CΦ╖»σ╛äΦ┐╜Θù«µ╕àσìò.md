# C 路径追问清单（有现成材料时）

用户贴了文档/SOP/流程图/历史样本时用。

**流程**：
1. AI 先完整读完用户提供的材料
2. 用下面 8 项识别清单扫一遍
3. 缺 ≥3 项 → 启动追问，最多 5 轮
4. 5 轮后还说不清 → 退回 A 路径从头访谈

---

## 8 项必备信息识别 + 追问话术

| 必备信息 | 怎么判断缺失 | 追问话术 |
|---|---|---|
| **最终交付物的具体形态** | 没有具体名词（如"报告""客户回复"），只有动词 | "这件事做完，最终交付的东西是什么？比如是一个 PDF、一封邮件、还是一个状态变更？" |
| **验收标准** | 没有"好"的衡量方法，只有"做完" | "你怎么判断这件事做得够好了？给我一个能观察的标准——点击率、客户回复、老板签字……" |
| **最终消费者** | 只提了执行人和验收人，没提受益人 | "谁是最终消费者/受益者？可能和验收人不一样。验收人在意的事和消费者真正需要的，是不是一回事？" |
| **频次** | 没说多久一次 | "这件事是一次性的，还是每天/每周/每月做？" |
| **上下游** | 没说前置和后置 | "做这件事之前，谁/什么会触发它？做完之后，结果交给谁/进入哪个流程？" |
| **现有工具** | 没说用什么做 | "现在你用什么工具做这件事？Office、飞书、自己写的小脚本……都说说。" |
| **痛点** | 全是中性流程描述，没有"卡点""慢""错""烦" | "这件事里最让你烦 / 最容易出错 / 卡过最久的是哪一步？" |
| **不可动项** | 没说约束 | "有什么是绝对不能让 AI 碰的？合规、保密、关系层面？" |

---

## 追问轮次控制

- 一次最多问 3 个问题，不要一上来 8 题连发
- 用户答完一轮 → 重新扫识别清单 → 决定下一轮问什么
- 5 轮还有 ≥3 项缺失 → 说："你提供的材料和回答里还缺这几项关键信息：[列出]。这条活儿先用现有信息做粗审计，标 `[待补]`，等你回头补上再深入。或者我们换条更清楚的活儿先拆。"

---

## 材料阅读清单

读用户给的材料时，重点抓这些信号：

| 信号 | 在哪找 |
|---|---|
| 流程步骤 | 标题、目录、编号列表 |
| 工具 | 截图、链接、产品名 |
| 数据/指标 | 表格、KPI 数字 |
| 卡点 | "需要……才能继续"、"等……" |
| 不可动项 | "必须……"、"不能……"、"由 X 负责" |
| 频次 / 时间 | "每周一"、"季度末"、"X 小时" |

读完后用 [A 路径访谈脚本.md](A路径访谈脚本.md) 第 1-5 段的字段表对照，**已经从材料里拿到的字段不要再问**。

---

## 反模式

1. ❌ 没读完材料就开始问（用户最烦"你不是已经看过了吗")
2. ❌ 一次性丢 8 个问题（信息量过载，用户会随便答）
3. ❌ 把材料里明显写了的内容当成"用户没说"（说明 AI 没认真读）
4. ❌ 5 轮追问还在问表面问题（说明问的方向错了，不是问得不够）
