# AI-First 红灯清单方法论（v1.0 新增）

> 红灯不是"做不到的清单"，是"现在做不到、但值得盯的关键缺口清单"。
> 红灯跨路径来源：路径 A 演进时搜不到方案 / 路径 B 重构时炸出"AI 还差一口气"的关键缺口。

## 为什么独立成一个文件

v0.4 把"红点"埋在 AI First 重做章节里，余一明确反馈：
- 应改名"红灯"——更醒目
- 应跨路径——不只是路径 A
- 应更详细——每条要给"自建路径 + 等待信号"
- 应独立文件——容易回看，不被淹没

## 红灯的两种来源

### 来源 A：演进路径上"现在做不到"

触发信号：
- 某原子的 subagent 搜索结果是"未找到符合标准的方案"
- 某原子评估为"AI 能力等级 = 不能 / 部分能-辅助"，且人机分工偏"人主"
- 某原子在 [snapshots/ai-capability-snapshot.md](../snapshots/ai-capability-snapshot.md) 对应的能力维度被标"待校准"或"不成熟"

含义：在不改变流程的前提下，AI 帮不上忙的关键节点。

### 来源 B：重构路径上"差一口气"

触发信号：
- 路径 B 重构出的新流程里，存在一个或多个"如果 AI 能做到 X，整件事就完全自动化了"的关键节点
- 这个 X 现在是 AI 做不到 / 做不稳的

含义：从目标倒推，AI 能力的临界缺口——补上这一口，整套重构方案的杠杆陡增。

**来源 B 比来源 A 更宝贵**——前者是"现状的限制"，后者是"未来的赌点"。

## 一条红灯的标准格式

```markdown
### 红灯 N：[一句话描述缺口]

- **来源**：路径 A / 路径 B / A+B 都触发
- **触发原子或节点**：[原子名 / 重构方案中的某节点]
- **缺口具体是什么**：
  [具体描述 AI 当前做不到的是什么——不要"AI 不够好"这种模糊话]
  [量化：当前能做到 X 水平，需要做到 Y 水平]

- **为什么是关键**：
  [说明这个缺口堵住了什么——一条工作流的什么节点 / 自动化链的什么环节]
  [量化杠杆：补上后能省什么 / 解锁什么]

- **自建可能路径**（如果不等行业方案）：
  - 路径 1：[用什么现有积木 + 拼接方式]，预期能达到 [X 水平]，离目标差 [Y]
  - 路径 2：…
  - 评估：[值得自建吗 / 等行业方案]

- **等待信号**（如果选择等）：
  - 信号 1：[什么样的产品发布算这个缺口被解决] —— 比如「某模型在 X 任务上跑分超过 Y」
  - 信号 2：[什么样的开源项目算这个缺口被解决]
  - 信号 3：[什么样的 benchmark 数据算这个缺口被解决]
  - 监测建议：把这些信号加到周报扫描里 / 关注 X 个账号

- **临时绕道方案**（在缺口被补上之前怎么办）：
  - 方案：[人继续做这一步 / 用次优工具 / 改流程绕开]
  - 代价：[这样做的成本和质量代价]
```

```markdown
# AI First 红灯清单

> 版本：v1.0 | 时间：YYYY-MM-DD | 关联活儿：[活儿名]

## 红灯总览

| # | 来源 | 缺口 | 关键性 | 自建可能 | 推荐行动 |
|---|---|---|---|---|---|
| 1 | A | … | 高 | 中 | 等下一代 |
| 2 | B | … | 高 | 高 | 自建 |
| 3 | A+B | … | 中 | 低 | 等待 + 监测 |

（按"关键性 × 自建可能"排序，越上面的越值得动手）

## 红灯详细描述

### 红灯 1：…
[完整格式]

### 红灯 2：…

### 红灯 3：…

## 总结：本活儿的 AI 临界缺口图

[一段话总结：这个活儿当前最值得盯的 1-2 个红灯是什么，为什么]
[可视化：mermaid 图，标出整套流程里红灯节点的位置]
```

## 关键性判断（高/中/低）

- **高**：补上这一灯，整条流程自动化率从 X% 跳到 Y%（≥ 30 个百分点）
- **中**：补上这一灯，能解锁 1-2 个原子的"人 → AI"切换
- **低**：补上这一灯，只是质量或速度边际改善

## 自建可能（高/中/低）

- **高**：用现有 LLM + Prompt + 已有工作区积木 1-2 周内可拼出 70% 水平的版本
- **中**：可拼，但需要数据收集 / 标注 / 多轮调试
- **低**：核心瓶颈在模型能力本身，自建意义不大，建议等

## 推荐行动（枚举）

- 自建（高自建可能 + 高关键性）
- 等下一代（低自建 + 高关键性 + 等待信号清晰）
- 等待 + 监测（中关键性，定期看进展）
- 临时绕道（低关键性，不值得追）
- 接受（这一灯短期补不上，接受人来做）
