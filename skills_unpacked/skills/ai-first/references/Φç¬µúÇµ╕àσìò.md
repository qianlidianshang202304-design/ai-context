# 自检清单

每次跑完一次都过一遍。任何一项打 ✗ 都不能交付。

---

## 通用自检（所有入口都跑）

### 起手

- [ ] 是否问了用户选 A/B/C 入口
- [ ] 是否问了"有没有现成材料"
- [ ] 是否引用了 [snapshots/ai-capability-snapshot.md](../snapshots/ai-capability-snapshot.md)，且快照 ≤ 7 天

### 访谈

- [ ] 用对话感问，不机械念脚本
- [ ] 第 3 段同时拿到了"验收人"和"最终消费者"，识别了错位（如有）
- [ ] 第 2 段拿到了"协作依赖"和"等待断点"
- [ ] 5 段必备字段全部填上或标 `[待补]`
- [ ] 标 `[待补]` 的字段 ≤ 2 个（> 2 个不允许进阶段 2）
- [ ] 如果用了推理填充模式：每个字段标注了 [用户原话]/[AI 推理]/[待确认]
- [ ] 如果用了推理填充模式：用户确认过推理版本后才进入阶段 2

### 原子拆解（v1.0 加第 4 门）

- [ ] 每个原子都过了**四门**（输入/输出/可评估/**工具映射**）
- [ ] 原子粒度一致（每个原子能映射到一次主工具调用）
- [ ] 没有"做研究""思考方案""写初稿"这种说不清工具映射的伪原子

### 评估（v1.0 升 8 维，v1.2 增分级）

- [ ] 评估分级表在 03 开头（Tier 1/2/3 各包含哪些原子 + 分级理由）
- [ ] Tier 1 原子：**8 个维度**都打了标（含第 8 维"验证步骤"）
- [ ] Tier 2 原子：4 维卡片 + 验证步骤，标注推理依据
- [ ] Tier 3 原子：总览行 + 分级理由 + "不能"的进红灯
- [ ] 每个维度的取值都在枚举范围内（没自由发挥）
- [ ] 风险维度至少在"准确性 + 合规 + 关系"三项上明确取值（Tier 1 必须，Tier 2 给概要）
- [ ] **第 8 维验证步骤**是"5 分钟可跑的具体动作"，不是"试用看看"（Tier 1+2 必须有）

### 市面方案搜索（v1.0 强制 subagent 并行，v1.2 增降级链）

- [ ] **派发 subagent 并行**搜索（不是串行调用 web-access）
- [ ] 一次消息内并行派发全部原子的搜索任务
- [ ] 每个 subagent 产物含：推荐 + 备选 + URL + 日期 + 中英文搜索词 + ≥1 条淘汰名单
- [ ] 推荐方案在过去 3 个月内有更新（或已标注超期已验证）
- [ ] "暂无成熟方案"的原子已标记并写入 `04_AI-First重做.md` 红灯清单（来源 A）
- [ ] **v1.2 降级链检查**：如果在 L2/L3 模式下执行——
  - [ ] 00_概要.md 显著标注执行模式（L2/L3）
  - [ ] L3 模式下无 URL（URL 是搜索产物，推理时禁止）
  - [ ] L3 模式下每个推理方案附验证步骤（搜索词 + 验证方法 + 判断标准）
  - [ ] 红灯清单含"方案搜索未执行"条目
  - [ ] MVP 第一步为"补跑方案验证"

### AI First 重做（v1.1 真方案）

- [ ] 路径 A 演进式 + 路径 B 重构式都给了
- [ ] **路径 A 含**：完整流程图（mermaid）+ 工具映射表 + 第一周 5 步 + 关键指标对比
- [ ] **路径 A 增补审视**：显式回答了"AI 介入后应补回哪些原来省略的步骤"（标 ✚），或明确写"无增补需求 + 理由"
- [ ] **路径 B 含**：真目标重定义 + 新架构图（mermaid）+ 关键模块设计 + 迁移路径分阶段 + 关键缺口标记
- [ ] 三档颗粒度（动作级 / 流程级 / 目标级）都有具体内容
- [ ] 出了两路径对比表（在 00_概要.md）
- [ ] 给了"人的不可替代价值"
- [ ] `04_AI-First重做.md` 内含 MVP 清单（3-5 个具体动作）
- [ ] **重构版自动化率 vs 演进版差距 > 10 个百分点**

### 红灯清单（v1.0 新增）

- [ ] 至少 1 条红灯（路径 A 来源 或 路径 B 来源）
- [ ] 每条红灯有"自建路径 + 等待信号 + 临时绕道"三段
- [ ] 红灯总览按"关键性 × 自建可能"排序
- [ ] 红灯节点在新架构图里标了 ⚠️
- [ ] 红灯监测计划落到工作流（不是写完就忘）

### 步骤文件输出（v1.1）

- [ ] 输出目录：`02AI 余一/工作流审计/[活儿名]_[YYYY-MM-DD]/`
- [ ] 5 个步骤文件全部落盘（00_概要 ~ 04_AI-First重做）
- [ ] 没有 per-atom 评估文件或搜索结果文件
- [ ] `03_评估与市面方案.md` 同时包含每个原子的 8 维评估和市面方案
- [ ] `04_AI-First重做.md` 同时包含演进版、重构版、红灯、MVP
- [ ] 00_概要.md 是最后写的，汇集了 01-04 的核心结论
- [ ] 跨文件引用都是相对路径，可点开

### 真对话约束（v1.0 新增）

- [ ] 访谈一次只问 1-2 题，等用户真回答
- [ ] 没有 AI 自填字段（即使在测试模式也不行）
- [ ] 没有跳段，5 段都跑了
- [ ] 模糊回答都追问过 1 次以上

## 反作弊自检

AI 偷懒的常见模式，每次都扫一遍：

- [ ] 没在评估时凭记忆吐"我记得有个工具叫……"
- [ ] 没把"通用 LLM 都能做"当成方案
- [ ] 没把厂商 PPT 上画的能力当成已交付方案
- [ ] 没把 1 年前看过的产品当成"最新方案"
- [ ] 没把"调研一下""学习一下"当成 MVP 动作
- [ ] 没把"想清楚""沟通好"当成原子的输出
