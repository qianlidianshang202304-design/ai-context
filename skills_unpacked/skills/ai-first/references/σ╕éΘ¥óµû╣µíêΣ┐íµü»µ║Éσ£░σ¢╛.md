# 市面方案信息源地图

不同类型的原子，去不同的源。**不要全部靠 Google 一把梭。**

---

## 信息源分层表

| 原子类型 | 主源 | 副源 | 不去的源 |
|---|---|---|---|
| **通用 LLM 能力**（写、读、总结、翻译、问答、分类） | OpenAI/Anthropic/Google/DeepSeek/通义/智谱 等厂商官方 changelog 和模型卡 + Artificial Analysis 榜单 | LMArena、SWE-bench、MMLU 等垂直榜 + Hacker News 高赞讨论 | 自媒体软文、营销公众号 |
| **垂直 AI 工具**（设计、视频、客服、销售、招聘等） | Product Hunt 近 3 个月榜 + a16z Top 100 GenAI Apps | 行业 newsletter（Ben's Bites / The Rundown / 量子位） + 用户真实评测 | 综合性"100 个 AI 工具大全"列表文 |
| **Agent / 工作流自动化** | GitHub Trending（Python/TypeScript，Star >1k 且近 3 月有 commit）+ LangChain / LlamaIndex / CrewAI / AutoGen 等生态 | Reddit r/LocalLLaMA、r/LangChain；HN Show HN 高赞 | 没代码的"Agent 平台"广告 |
| **企业级方案**（SaaS、私有化、咨询） | Gartner / Forrester 报告（看趋势不看排名）+ 厂商官方 case study + a16z / Sequoia 投资笔记 | 头部咨询公司（McKinsey / BCG / Deloitte）AI 实践文 | 厂商自家公众号；销售型白皮书 |
| **国内特化**（中文、合规、私有化、本地化） | 量子位、机器之心、智东西、36 氪 AI 板块 + 阿里云/腾讯云/火山引擎/华为云 案例库 | 各厂商技术博客 + 国内 GitHub Trending（中文项目）+ 即刻、小红书、知乎高赞产品体验 | 抖音"AI 教程"博主、付费课程引流文 |

## 选源四条原则

1. **官方优于二手**：能看官方 changelog 就不看自媒体复述
2. **代码优于宣传**：能看 GitHub commit 就不看 Landing Page
3. **第三方评测优于厂商自吹**：找对比测评而非单方面数据
4. **更新日期 > 文章质量**：6 个月前的"神文"也要重新验证

---

## 中英文双语必做

每个原子的搜索都要：
- 至少 1 次英文关键词搜索（覆盖国际方案）
- 至少 1 次中文关键词搜索（覆盖国内方案 + 本土化版本）

中文搜词参考：
- 工具类："[动作] AI 工具"、"[动作] 智能"
- Agent 类："[动作] 自动化"、"[动作] 智能体"
- 企业类："[行业] AI 解决方案"、"[行业] 数字员工"

英文搜词参考：
- "AI tool for [action]" / "best [action] AI 2026"
- "[action] agent open source"
- "[industry] AI automation"

---

## 不接受的"市面方案"

下面这些扔进推荐栏 = skill 失败：

- "ChatGPT 可以做"（太泛，不是方案）
- "用 GPT-4 写个 Prompt"（这不是方案，是默认能力）
- 没有产品名 / 没有 URL 的"听说有个工具能做"
- 已经停服 / 排队中 / Coming Soon 的产品
- 厂商 PPT 上画的能力但用户实际用不到的
- 半年以上没更新的开源项目（除非用户明确要"稳定"）

---

## 搜索成本控制

完整的市面方案搜索按原子算，每个原子大约：
- 3-5 次 web-access 调用
- 5-10 个候选过筛
- 1-3 个最终推荐

如果一条活儿有 10 个原子，市面方案搜索环节约 30-50 次调用。这是合理预算，**不要为了省成本砍掉搜索环节**——这是 skill 的核心价值。
