# 个人模式三层结构

完整入口（B）走这个。轻量入口（A）只走 L2。

---

## L0：人物画像（一次性建立，长期复用）

**为什么需要**：单条活儿的评估不能脱离"这个人是谁"。同一条"写公众号"对自由职业者和企业内容编辑，AI 介入边界完全不同。

**问什么**（按 [templates/人物画像.md](../templates/人物画像.md) 的字段问）：
- 职业身份 / 工作主线
- 一周时间分配（大概各类事各占多少时间）
- 现在同时在跑几条工作线
- 这个人的"不可替代价值"在哪（自我认知 + 别人评价）
- 体力/精力节律（什么时候高能、什么时候低能）
- 已有 AI 使用基础（用过哪些工具、Prompt 经验）
- 对 AI 的边界（什么愿意让 AI 做、什么不愿意）

**时间**：~15 分钟。

**输出**：`02AI 余一/工作流审计/[人名]_人物画像.md`

**复用**：之后每次新拆活儿都先读这个画像，不重新问。

---

## L1：活儿清单（这个人手里所有反复在做的事）

**为什么需要**：不列全 = 不算摊开。AI First 重做的最大价值之一是发现"原来这件事可以不用做"，但前提是看到全图。

**怎么列**：

按"活儿来源"扫一遍：
- 主业相关：哪些是主业职责必须做的？
- 副业相关：有没有副业 / 第二事业线？哪些活儿？
- 关系维护：客户、合作伙伴、社群、向上向下管理
- 自我成长：学习、复盘、记录、健身
- 家庭 / 生活：定期采买、家务安排、孩子（如有）
- 其他反复在做的：每月每周每天的固定动作

**对每条活儿打标**：

| 字段 | 取值 |
|---|---|
| 活儿名 | 一句话 |
| 频次 | 每天 / 每周 / 每月 / 每季度 / 不定期 |
| 单次时长 | < 30 分钟 / 30-120 分钟 / 半天 / 一天以上 |
| 重要性 | 高（核心职责）/ 中（必要支撑）/ 低（可外包） |
| AI 化潜力（粗判） | 高 / 中 / 低（粗判，详细评估在 L2） |
| 烦躁指数 | 高（很烦）/ 中 / 低（不讨厌甚至享受） |

**时间**：第一次 ~30 分钟。**允许分多次补**，因为人想不起来全部活儿很正常，建议跑 1 周后回来补漏。

**输出**：`02AI 余一/工作流审计/[人名]_活儿清单.md`

**优先级排序**：
- AI 化潜力高 + 烦躁指数高 + 频次高 = **优先拆解**（高 ROI）
- AI 化潜力低 + 重要性高 + 享受 = **不拆**（这是人的护城河）
- AI 化潜力中 + 频次低 = 后置

---

## L2：单条活儿审计（按优先级逐条拆）

走 [SKILL.md 的"单条活儿审计流程"](../SKILL.md#单条活儿审计流程)。

**节奏**：每条是一个独立的"步骤文件单元"（5 个主文件）。**允许分多次跑**，每次完成一个或几个阶段（参考 SKILL.md 的 8 阶段）。

**输出**：`02AI 余一/工作流审计/[活儿名]_[YYYY-MM-DD]/`（v1.1 起每条活儿是步骤文件目录，不按原子建子文件）

**节奏建议**：
- 第一条挑"高 ROI"那条，跑通体验
- 第二条挑"暂无成熟方案"概率高的（探索性的），看 AI First 视角能不能炸出新东西
- 第三条挑"不可动项多"的（约束密集的），测试边界
- 跑完 ≥3 条，进 L3

---

## L3：全人 AI First 合成（核心交付物）

**为什么是核心**：单条审计是中间产物。L3 才是这个 skill 真正交付的东西。

**触发条件**：≥3 条单条审计完成。

**做什么**（详见 [全人合成方法论.md](全人合成方法论.md)）：
1. 横向扫描所有审计报告，识别共性原子（可以被同一个 AI 助手 / 工作流通吃的部分）
2. 识别可消失的活儿（AI First 视角下根本不需要做了）
3. 识别可合并的活儿（多条变一条）
4. 识别这个人重新设计后的"一周"长什么样
5. 识别这个人的不可替代价值集中在哪几个原子
6. 给出 AI First 重做后的"新一周时间分配 vs 现在"对比

**输出**：`02AI 余一/工作流审计/[人名]_全人AI-First重做_[YYYY-MM-DD].md`

按 [templates/全人AI-First重做报告.md](../templates/全人AI-First重做报告.md) 模板。

---

## 整体节奏（结构性指标，不预估时间）

| 入口 | 第一次跑通 | 后续每次 |
|---|---|---|
| 轻量入口（A） | 单条活儿（5 个步骤文件） | 单条活儿（5 个步骤文件） |
| 完整入口（B） | L0 + L1 + L2 ≥3 条 + L3 一份 | 加 L2 一条 / 重跑 L3 |
| 企业入口（C） | 不允许直接进，必须 ≥3 个人各完成一份全人报告 | — |

完整入口建议**分多次完成**，单次完成"一个独立单元"：
- 单元 1：L0 人物画像（一次写完）
- 单元 2：L1 活儿清单（一次列完，可后续补）
- 单元 3 ~ 单元 N：每个单元跑一条 L2（5 个步骤文件）
- 收尾单元：跑 L3 全人合成

> 不预估具体时间——AI 不懂时间，余一的真实节奏只有余一自己知道。结构性指标是"几个单元、每单元几个文件"。

---

## 反模式

1. ❌ B 入口跳过 L0 / L1 直接拆某一条 → 失去全图
2. ❌ L1 只列了 5 条就开始拆 → 大概率漏了核心活儿，等 L3 时发现摊得不完整
3. ❌ L2 只拆"AI 化潜力高"的，不拆"不可动项多"的 → 不知道边界
4. ❌ 跑完 3 条 L2 不出 L3 → 这个 skill 的核心价值没兑现
5. ❌ L3 只是"3 份审计的拼贴"，没做合成判断（消失/合并/重组）
