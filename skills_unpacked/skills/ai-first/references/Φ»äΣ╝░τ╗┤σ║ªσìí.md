# 评估维度卡（8 维度，v1.0）

每个原子按这 8 个维度打标签。**取值必须落在下面给出的枚举范围内，不允许自由发挥**——否则跨原子无法对比。

v1.0 比 v0.4 多了第 8 维「验证步骤」，回应余一"用户怎么实操验证 AI 真能做到"的诉求。

---

## 维度 1：AI 能力等级

| 取值 | 含义 |
|---|---|
| 不能 | 当前 AI 完全做不到，或质量远低于及格线 |
| 部分能-辅助 | AI 能给出参考，但必须人主导 + 人最终决策 |
| 部分能-需校验 | AI 能完成主要工作，但产出必须经人审核才能交付 |
| 能-可独立 | AI 可以独立完成并直接交付，人只做抽检 |
| 超过人类 | AI 在质量、速度、一致性等关键指标上已超过该领域熟练人类 |

**判断锚点**：参考 [snapshots/ai-capability-snapshot.md](../snapshots/ai-capability-snapshot.md)，不凭印象。

---

## 维度 2：预期质量

格式：一句话描述 AI 能做到的水平 + 对标人类档位。

**人类档位枚举**：
- 初级（入行 < 1 年）
- 熟练（入行 1-3 年）
- 资深（入行 3-7 年）
- 专家（领域顶尖）

**示例**：
> "AI 写出的初稿质量约等于熟练写作者一稿水平，距离资深还差一次结构调整和 1-2 个深度案例补充。"

---

## 维度 3：人机分工

| 取值 | 含义 |
|---|---|
| 人独立 | AI 完全不参与（通常配合"AI 能力等级=不能"） |
| 人主 + AI 辅 | 人做核心判断，AI 提供候选/资料/初稿 |
| 人机并行 | 人和 AI 各做一部分独立产出，最后合并 |
| AI 主 + 人审 | AI 完成主要工作，人审核校准 |
| AI 独立 | AI 完成 + 自动质检，人只做抽检 |

---

## 维度 4：市面最新方案

格式：每个原子至少 1 个推荐 + ≤2 个备选。

**每个推荐必须包含**：
- 方案名（产品/工具/框架/Prompt 模式）
- 提供方（厂商/开源社区/独立开发者）
- 一句话能力描述
- 适用边界（什么场景能用，什么场景不行）
- 上手成本（API 调用 / 注册即用 / 需自建 / 需采购）
- **来源 URL** + **访问日期**（缺一不可）

**搜索方式**：强制走 [搜索方案subagent调用规约.md](搜索方案subagent调用规约.md)，subagent 并行跑。

---

## 维度 5：推荐路径

| 取值 | 含义 |
|---|---|
| 现在就能用 | 当周可上手，方案成熟 |
| 短期内可用 | 工具尚在快速迭代，建议先小范围试 |
| 等下一代 | 当前方案能力或稳定性不够 |
| 自建 | 现成方案不匹配，需要基于通用 LLM + Prompt/Agent 自己拼 |
| 外包 | 不值得自己做，找服务商 |
| 暂不推 | 风险或成本不划算 |

---

## 维度 6：成本

格式：分两层。**金额单位用 ¥ 或 $，同一份报告内保持一致。**

**单次执行成本**：
- 调用费：[金额] / 次（API 费用，没有就写"零"）
- 人力消耗：低 / 中 / 高（不预估具体时间，因为 AI 不懂时间——给"低/中/高"档）
- 隐性成本：是否需要数据清洗/标注/Prompt 调试

**规模化后成本**：
- 按用户实际频次计算月度总成本
- 与"完全用人"的对比：成本降低 X% / 差不多 / 更贵

---

## 维度 7：风险

每个原子必须给出 ≥1 个风险维度的判断。**不能写"无风险"**——一定要列出来再判断为"低"。

| 风险维度 | 取值 |
|---|---|
| 准确性风险 | 高/中/低（AI 出错的代价多大） |
| 合规风险 | 高/中/低（数据隐私 / 监管 / 行业规则） |
| 关系风险 | 高/中/低（用 AI 让对方知道后是否影响信任） |
| 替代风险 | 高/中/低（这步外包给 AI 后，自己核心能力是否被掏空） |
| 失控风险 | 高/中/低（AI 自动跑出错没人发现的概率 × 后果） |

**至少在"准确性 + 合规 + 关系"三项上明确取值**。

---

## 维度 8：验证步骤（v1.0 新增）

> 用户拿到这张评估卡，要能自己跑一个"5 分钟可完成的盲测/对比/质检"，验证 AI 真的能做到声称水平。

**格式**（具体动作，不抽象）：

```
1. 准备：[需要什么样本/输入]
2. 操作：[让 AI 做什么]
3. 比对：[和什么对比]
4. 通过标准：[什么样的结果算 AI 能做到声称水平]
5. 不通过怎么办：[降级到哪个能力等级 / 调整 Prompt / 换工具]
```

**示例 1（"AI 用余一风格写一段 400 字"）**：

```
1. 准备：3 段余一近期文字（每段 200-400 字）作为风格样本
2. 操作：给 AI 喂 3 段样本 + 主题 X，要求出 3 个版本初稿
3. 比对：把 3 个 AI 版 + 1 个余一真笔的版本混在一起，找朋友盲读
4. 通过标准：朋友 5 次盲测里"猜错"次数 ≥ 2 = AI 能做到熟练写作者水平
5. 不通过怎么办：
   - 错认 0-1 次 → 降级到"部分能-辅助"，AI 只做参考不直接用
   - 调整方向：增加风格样本到 10 段；或在 Prompt 里加"内心 OS 显性化"等具体指令
```

**示例 2（"从 N 个信息源筛选 K 条相关信息"）**：

```
1. 准备：N=20 个最近读过的信息源 URL + 主题"AI Coding 工具新进展"
2. 操作：让 AI 筛出 K=5 条最相关
3. 比对：自己也筛 5 条，看重叠度
4. 通过标准：重叠 ≥ 3/5 = AI 能做到熟练助理水平
5. 不通过怎么办：
   - 重叠 0-2 → 主题描述太模糊，加 3-5 个明确"我关心的子方向"
   - 重叠 3-4 → 通过，但保持人审
```

**反例**（不合格的验证步骤）：

- ❌ "试用一下看看效果" → 太抽象
- ❌ "和人写的对比" → 没说怎么对比、什么算通过
- ❌ "如果不好就重新调" → 没有明确不通过路径

## 8 维度卡片格式（最终输出）

每个原子最终输出一张卡片：

```markdown
### 原子 N：[原子名]

- 输入：…
- 输出：…
- 成功标准：…
- 工具/Prompt：…

| 维度 | 取值 |
|---|---|
| AI 能力等级 | [枚举] |
| 预期质量 | [一句话 + 人类档位] |
| 人机分工 | [枚举] |
| 推荐路径 | [枚举] |

**市面方案**（来自 subagent 并行搜索）
1. [推荐] 方案名（提供方）— 能力描述 — 适用边界 — 上手成本 — [URL @ YYYY-MM-DD]
2. [备选] …

**成本**
- 单次：API [金额] + 人力消耗 [低/中/高]
- 月度（按 N 次/月）：[金额]，相比纯人力 [对比]

**风险**
- 准确性：[低/中/高] [一句话]
- 合规：[低/中/高] [一句话]
- 关系：[低/中/高] [一句话]
- 其他：…

**验证步骤**（5 分钟可跑）
1. 准备：…
2. 操作：…
3. 比对：…
4. 通过标准：…
5. 不通过怎么办：…
```
