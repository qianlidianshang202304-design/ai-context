# 原子拆解标准

## 定义

> **一个原子 = 一个有明确输入、明确输出、明确成功标准、且可一句话说出该用哪个工具/Prompt 来做的动作。**

四要素缺一 = 还没拆到底。

## 四条硬门（v1.0 起，比 v0.4 多了第 4 门）

每个候选原子必须**同时**通过：

1. **输入门**：能写出"开始这一步前手里有什么"。模糊的"有了一些信息" / "有点想法"不算。
2. **输出门**：能写出"做完这一步后产出了什么具体的东西或状态"。"想清楚了"不算，"写下来了一段 200 字的核心论点"算。
3. **可评估门**：能写出"怎么判断这一步做得够好"。一个可观察的标准，不是"凭感觉"。
4. **工具/Prompt 映射门** *（v1.0 新增）*：能用一句话说出"这一步用什么工具/什么 Prompt 模式 / 什么模型 / 什么 SOP 来做"。
   - 通过例：「用 Claude + Prompt 模板《风格守卫》+ 喂 5 篇余一往期文 + 写主题 X」
   - 不通过例：「用 AI 来做」「找个工具帮忙」「让模型生成」
   - 通过这一门 = 可以直接派给 subagent 去搜方案、可以直接对应到一次工具调用

## 为什么加第 4 门

v0.4 测试时"写初稿"被当成原子，但它实际包含：
- 拆小标题
- 每节 hook
- 段落扩写
- 段间过渡
- 节奏标注
- 重写

每一步对应不同的 Prompt / 工具调用。原子级 = 能直接编排成工作流的最小单元。

## 反例库

| 反例 | 问题 | 怎么改 |
|---|---|---|
| "做研究" | 太大，过不了任一门 | 拆成"列研究问题 / 搜信息源 / 筛 N 条 / 归纳 / 写"等多个原子 |
| "用 AI 写文章" | 过不了第 4 门——没说哪个 AI、什么 Prompt、什么输入 | 拆成"AI 列大纲 / 人定大纲 / AI 出初稿 / 人改 N 处 / AI 润色"，每步都能映射工具 |
| "写初稿" | 过不了第 4 门——一次工具调用做不完 | 拆成"生成 N 条小标题 / 每节 1 句 hook / 第 1 节扩写 / 第 2 节扩写 / 全文通读节奏标注 / 针对标注重写 N 段" |
| "思考方案" | 输出和工具都说不清 | 改为"写下 3 个候选方案 + 各自理由 + 推荐 1 个"，工具：Claude + 决策矩阵 Prompt |
| "和客户沟通" | 输入输出都模糊 | 改为"获取客户对 X 问题的明确答复"或"让客户对方案签字" |
| "整理资料" | 输出和工具都说不清 | 改为"把 N 个素材按 3 个标签分类，每标签下按时间倒序"，工具：脚本或 LLM 标注 |

## 正例库

| 正例 | 输入 | 输出 | 成功标准 | 工具/Prompt |
|---|---|---|---|---|
| 从 N 个信息源筛选与主题相关的 K 条 | 信息源 URL 列表 + 主题 | K 条筛选后信息（含来源摘要） | 相关性人工抽检 ≥ 80% | scout-agent + Prompt《主题相关性筛选 v1》 |
| 把 K 条信息归类为 3-5 个主题簇 | K 条信息 | 主题簇列表（每簇有名字+成员） | 簇间互斥、每簇 ≥3 条 | Claude + Prompt《主题聚类 v1》 |
| 对每个主题簇写 200 字摘要 | 主题簇+成员信息 | 每簇 200 字摘要 | 含核心观点+关键数据+典型案例 | Claude + Prompt《主题摘要 v1》 |
| 回复客户邮件确认会议时间 | 客户原邮件+自己日程 | 一封发出去的邮件 | 客户回复确认或自动接受 | Gmail/Outlook + Calendar API + Prompt《商务邮件 v1》 |
| 给一段录音做转写并标注 N 个金句 | 1 小时录音 | Markdown：全文转写+N 条金句标注（含时间戳） | 金句"独立可引用+体现观点+≤50 字" | Whisper + Claude + Prompt《金句萃取 v1》 |

## 拆解流程

1. 写下整件事的目的和最终交付物（一句话）
2. 倒推：要拿到这个交付物，前面必须先有什么？反复问"再前面呢"，直到追到起点
3. 每个候选步骤过四门（输入/输出/可评估/工具映射）
4. 过不了门的步骤继续往下拆
5. 检查粒度一致性：所有原子的"复杂度"应该在同一数量级——粗判方法是"一次工具调用 / 一次人判断"

## 颗粒度判断（结构性，非时间性）

| 类型 | 一个原子的复杂度 |
|---|---|
| 机械型（数据处理、查找、整理、发送） | 一次工具调用 / 一次脚本运行 |
| 信息型（搜索、阅读、提取、归类） | 一次或多次同模式 LLM 调用 |
| 判断型（选择、评分、决策） | 一次人判断 + 可选 AI 候选 |
| 创作型（写、设计、构思） | 一次主 Prompt 调用 + 必要的迭代 |

**判断脑力密集型 vs 机械型**：这个原子里"思考"占比多还是"动作"占比多。
- 思考占比 > 50% → 创作型/判断型，允许包含若干次 AI 迭代
- 动作占比 > 50% → 机械型/信息型，必须能压到一次工具调用

## 拆解过粗 / 过细的信号

**过粗**：评估时无法区分"AI 能 / 不能"——因为一个原子里既有 AI 能做的部分也有不能的。
**过细**：每个原子都是琐碎的鼠标点击/敲键盘动作，失去了"判断"的意义。

判断阈值（v1.0 强化）：**第 4 门是真正的颗粒度门**——
- 一个原子能不能映射到**一次主工具调用**？能 → 粒度合适
- 一个原子需要拆成多个不同 Prompt 才能做完 → 过粗
- 多个原子需要合起来才能给一个 AI → 过细
