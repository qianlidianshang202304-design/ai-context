# Team Match · 组队助手

## 产出物

读取「全场分享速记 + 自己的项目编号」，产出：

1. **组队推荐**：2-3 个最互补的项目 + 理由
2. **组队话术**：怎么发出邀请（一句话开场）
3. **团队 Build 建议**：合并方式、分工、60 分钟能做到什么、Demo 2 怎么讲

## 不负责

- 不替人做决定（给推荐和理由，最终人选）
- 不做开发（→ `ai-dev-sprint`）
- 不做展示（→ `demo-deck`）
- 不知道现在在工坊哪一段（→ `sprint-guide`）

## 输入契约

必需：
- 全场 Demo 速记 / 录音转写（所有项目）
- 自己的项目编号 + 自己做了什么

可选：自己想深化的方向、缺的能力、偏好团队大小。

缺速记时只问一件：「全场分享记录在哪、你是第几个项目」。

## 执行链

| 步  | 动作          | 产出                                            |
| --- | ------------- | ----------------------------------------------- |
| 1   | 解析全场      | 每个项目抽：做了什么 / 方向 / 能力 / 想组队意向 |
| 2   | 定位自己      | 按编号取出自己的项目画像                        |
| 3   | 互补匹配      | 用匹配维度打分，选 2-3 个                       |
| 4   | 出组队话术    | 每个推荐一句邀请开场                            |
| 5   | 出 Build 建议 | 合并方式 + 分工 + 60min 目标 + Demo 2 讲法      |

## 匹配维度

| 维度     | 问题                                       |
| -------- | ------------------------------------------ |
| 方向契合 | 两个项目能不能拼成一个更完整的东西？       |
| 能力互补 | 我缺的（设计/后端/业务/AI 工具链）他有吗？ |
| 意向匹配 | 他 Demo 里说想和谁组队，包含我这类吗？     |
| 杠杆叠加 | 合并后 AI 能产生更大杠杆吗？               |
| 规模适配 | 凑成 2-4 人小团队，决策快、Demo 聚焦       |

## 三种组队形态（给建议时分类）

| 形态 | 适用              | Build 重点                  |
| ---- | ----------------- | --------------------------- |
| 深化 | 我的项目有人接力  | 补设计 / 数据 / 部署 / 文档 |
| 合并 | 两个互补项目      | 工具 + 前端，脚本 + 可视化  |
| 新题 | Demo 激发的新想法 | 快速验证，仍要求可展示      |

## 输出格式

```text
## 你的项目（#__）画像
做了什么 / 方向 / 你的强项 / 你缺的

## 推荐队友
1. #__ 项目名 —— 互补点：__ ｜ 邀请开场：「__」
2. #__ ...
3. #__ ...

## 如果组成团队，建议这样 Build
形态：深化 / 合并 / 新题
分工：谁牵头 Demo，谁补 __，AI 负责 __
60 分钟目标：__
Demo 2 讲法：团队版多了什么 / AI 杠杆 / 24h 动作
```

## 验收

- [ ] 推荐基于全场速记，不是泛泛建议
- [ ] 每个推荐有具体互补理由 + 邀请话术
- [ ] Build 建议含分工 + 60min 目标 + Demo 2 讲法
- [ ] 形态判断明确（深化 / 合并 / 新题）
- [ ] 不替人拍板，保留选择权
