# AI 冲刺工坊 · Skills

参与者侧 skill 包。**进工坊先读 `sprint-guide`**，它会告诉你现在在第几段、用哪个 skill。
每个子目录是一个 skill：`SKILL.md` 是入口，`references/` 复制 SSOT 正文供离线使用。

```text
段 1 学习明确 30m → ai-first(挑活儿) → codex-goal(写 Goal)   [skill-onboarding 选读]
段 2 个人 Build 30m → ai-dev-sprint
段 3 第一次 Demo 60m → demo-deck
   茶歇 & 组队 15m → team-match
段 4 组队 Build 60m → ai-dev-sprint(基于反馈再优化) + team-match
段 5 第二次 Demo 30m → demo-deck
段 6 点评 Top 3      → pick-top3(主持人侧)
```

## skill 清单

| skill              | 段       | 干什么                                               | 出什么（第一个成果）    |
| ------------------ | -------- | ---------------------------------------------------- | ----------------------- |
| `sprint-guide`     | 全程     | 总入口/向导，按段路由，守「先出第一版再优化」节奏    | 知道下一步用哪个 skill  |
| `ai-first`         | 1        | 10 分钟挑一条活儿，做 AI-First 速判                  | 一张速写卡 + build 切片 |
| `codex-goal`       | 1        | 把切片写成可验证 Goal                                | Goal Brief              |
| `skill-onboarding` | 1        | 选读：理解 skill 是什么、怎么用好 AI                 | 现成 skill 体验         |
| `ai-dev-sprint`    | 2 / 4    | 时间盒内把 Goal 做成可展示成果；组队段基于反馈再优化 | 可运行成果              |
| `demo-deck`        | 3 / 5    | 读成果生成三页 demo + 分享策略                       | 三页 HTML demo          |
| `team-match`       | 茶歇 / 4 | 按全场速记推荐队友、给组队话术                       | 组队推荐 + Team Brief   |
| `pick-top3`        | 6        | 主持人侧：按三维选出值得继续的 Top 3                 | 打分表 + Top 3 推荐     |
| `skill-eval`       | 工具     | 评一个 skill 好不好（TRACE 五维），按需用            | 五维评估报告            |

## 贯穿原则：干中学 · 第一版优先

每一段都走同一个微循环，不要停在「想清楚再做」：

```
出第一版（哪怕粗糙）→ 拿它换反馈（AI 自检 / Demo / 队友）→ 基于反馈优化下一版
```

## 完整版与轻量版

- `ai-first/SKILL.md` 是**工坊轻量版**（10 分钟挑活儿）。
- 想做完整的全人 AI-First 审计（人物画像 / 原子拆解 / 8 维评估 / 搜方案 / 五文件报告），读同目录 `ai-first/ai-first-full-v4.0.md`，references/ 与 templates/ 配套不变。
